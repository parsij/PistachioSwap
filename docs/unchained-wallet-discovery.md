# Unchained Wallet Discovery

PistachioSwap derives wallet-token coverage from `ACTIVE_TOKEN_DISCOVERY_CHAINS` in the API token-discovery registry. Do not maintain a separate active-chain list.

Provider order for each wallet request:

1. self-hosted ShapeShift Unchained, when the chain has a verified upstream EVM coinstack and a configured local endpoint
2. existing wallet-token providers: Alchemy Portfolio, legacy Alchemy/Moralis/RPC paths
3. persisted curated fallback-token catalog, including native assets

Unchained, the fallback token catalog, and Gas Assist sponsorship allowlists are separate trust domains. A token discovered by Unchained or present in the fallback catalog is not automatically authorized for sponsorship, approvals, routing, or execution.

## Local Environment

```dotenv
UNCHAINED_ENABLED=true
UNCHAINED_REQUEST_TIMEOUT_MS=8000
UNCHAINED_HTTP_URLS_JSON={"1":"http://127.0.0.1:3101","10":"http://127.0.0.1:3110","56":"http://127.0.0.1:3156","100":"http://127.0.0.1:3100","137":"http://127.0.0.1:3137","8453":"http://127.0.0.1:3453","42161":"http://127.0.0.1:3161","43114":"http://127.0.0.1:3114"}
```

Localhost HTTP is accepted. Non-localhost HTTP remains rejected unless `UNCHAINED_ALLOW_INSECURE_HTTP=true` is explicitly set for a non-production local experiment.

## Manual Commands

```bash
pnpm unchained:print-start-commands
pnpm unchained:config:check
pnpm unchained:status
pnpm --filter @pistachio/api fallback-tokens:build
pnpm --filter @pistachio/api fallback-tokens:audit
```

For each upstream coinstack, work in a local ShapeShift Unchained checkout:

```bash
cd node/coinstacks/<coinstack>
cp sample.env .env
```

Fill only the required local secrets in that Unchained checkout, such as `INDEXER_API_KEY`, `RPC_API_KEY`, `WEBHOOK_URL`, and `ETHERSCAN_API_KEY` where the sample requires them. PistachioSwap diagnostics never expose those values.

## Hidden Tokens

Unchained confirms balances, not trust. Exact `chainId + lowercase contract address` identity is joined to trusted catalog metadata. Unknown, malformed, unverified, or provider-confirmed spam holdings remain recoverable in Hidden tokens and do not contribute to portfolio totals.
